# Scout

Scout is a broadband competitive analysis web app for Broadband / Communication Service Providers.
It turns a list of ZIP codes into a complete competitive briefing: who else sells broadband in the
footprint, their technology and speed mix, customer sentiment, recent launches, demographics, an
AI readiness score for the user's own marketing site, and a ranked list of opportunities for the
user's next move.

## Stack

- **Next.js 15** (App Router) on Vercel
- **Tailwind CSS**
- **Supabase** for persistence (optional; the app falls back to safe stubs when env vars are absent)
- **Cheerio** for AI Readiness HTML inspection

## Getting started

```bash
pnpm install
pnpm dev
# → http://localhost:3000
```

No environment variables are required to run the app — every external integration has a stub
fallback. To connect real data, copy `.env.example` to `.env.local` and fill in:

| Variable | Purpose |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon key |
| `CENSUS_API_KEY` | Optional — Census ACS API key for higher rate limits |

## Routes

- `/` — landing
- `/analyze` — ZIP entry form
- `/report/[slug]` — tabbed competitive briefing (Overview, Opportunities, Competitors, Coverage, Demographics, Launch radar, AI readiness)
- `/ai-readiness` — standalone AI readiness scanner
- `/api/analyze` — POST creates a report
- `/api/ai-readiness` — POST scores a URL

## Supabase schema

```sql
create table bdc_zip_provider (
  zip text, provider_name text, technology text,
  max_down_mbps int, max_up_mbps int, locations int,
  primary key (zip, provider_name, technology)
);
create table competitor_reviews (
  provider_name text primary key,
  stars numeric default 0, review_count int default 0,
  source text default 'google', updated_at timestamptz default now()
);
create table competitor_news (
  id bigint generated by default as identity primary key,
  provider_name text, title text, url text, published_at date,
  category text
);
create table reports (
  slug text primary key, zips text[], company_name text,
  created_at timestamptz default now()
);
```

Until these tables are populated, Scout ships with:

- A deterministic stub provider catalog driving overlap, tech mix, and max speeds per ZIP
- 0-star / 0-review placeholders for Google reviews
- A mock competitor news feed driving the Opportunities engine
- A live Census ACS lookup for demographics (falls back to stub on error)
